### What Happened and Impact

On 2026-03-02 at approximately 15:40 UTC, the `user-service` experienced a critical failure resulting in all API requests returning a 504 Gateway Timeout error. This led to a significant impact on user experience, with users unable to interact with the service for an extended period (30 seconds). The incident was severe due to its high severity and widespread effect.

### Root Cause Analysis

The root cause of this incident was identified as the addition of a query filtering users by `last_login_at` without indexing the corresponding column. This led to poor query performance, causing CPU usage to spike to 20% (normal range is 10-30%) and memory usage to reach 80% (normal range is 50-90%). The resulting high latency caused all API requests to timeout with a 504 Gateway Timeout error.

### Resolution Summary

To resolve the incident, two key steps were taken:

1. An index was added on the `last_login_at` column to optimize query performance and prevent similar issues in the future.
2. A review of API request handling logic revealed opportunities for improvement to prevent high latency under heavy loads.

### Prevention Recommendations

To prevent such incidents from occurring in the future, we recommend:

* Implementing automated checks for potential performance bottlenecks (e.g., indexing columns used in queries)

By following these recommendations, we aim to improve the overall reliability and responsiveness of our services.
