<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Wire Azure Monitor alerts (VM + App Service)</name>
  <action>Create Azure Monitor alert rules via CLI + Action Group to pagemenot webhook.</action>
  <instructions>
    All commands on LOCAL MACHINE. Use screen for each.

    **Step 1 — Create Action Group pointing to pagemenot:**
    ```bash
    screen -dmS az-ag bash -c "
      az monitor action-group create \
        --name pagemenot-webhook \
        --resource-group pagemenot-rg \
        --short-name pagemenot \
        --action webhook pagemenot-hook https://34.123.60.64:8080/webhooks/azure --use-common-alert-schema true \
        > /tmp/az-ag.log 2>&1
    "
    ```
    Check: `cat /tmp/az-ag.log | tail -3`
    Get action group ID: `az monitor action-group show -n pagemenot-webhook -g pagemenot-rg --query id -o tsv`

    **Step 2 — VM heartbeat alert (fires when VM is deallocated ~5min):**
    ```bash
    AG_ID=$(az monitor action-group show -n pagemenot-webhook -g pagemenot-rg --query id -o tsv)
    VM_ID=$(az vm show -g pagemenot-rg -n pagemenot-test-vm --query id -o tsv)

    screen -dmS az-vm-alert bash -c "
      az monitor metrics alert create \
        --name pagemenot-vm-heartbeat \
        --resource-group pagemenot-rg \
        --scopes '$VM_ID' \
        --condition 'avg Percentage CPU < 0' \
        --description 'VM stopped or unreachable' \
        --severity 1 \
        --window-size 5m \
        --evaluation-frequency 1m \
        --action '$AG_ID' \
        > /tmp/az-vm-alert.log 2>&1
    "
    ```
    Note: Azure doesn't expose heartbeat as a standard metric for CLI alerts. Use VM availability or CPU absence.
    If CPU-based alert doesn't fire reliably: in Azure Portal → Monitor → Alerts → Create:
      - Scope: pagemenot-test-vm
      - Signal: VM Availability (% availability < 100)
      - Window: 5 min, frequency: 1 min
      - Severity: Sev1
      - Action Group: pagemenot-webhook (common alert schema ON)

    **Step 3 — App Service availability alert:**
    ```bash
    APP_ID=$(az webapp show -g pagemenot-rg -n pagemenot-test-app --query id -o tsv)
    AG_ID=$(az monitor action-group show -n pagemenot-webhook -g pagemenot-rg --query id -o tsv)

    screen -dmS az-app-alert bash -c "
      az monitor metrics alert create \
        --name pagemenot-app-availability \
        --resource-group pagemenot-rg \
        --scopes '$APP_ID' \
        --condition 'avg Http5xx > 5' \
        --description 'App Service returning 5xx errors' \
        --severity 2 \
        --window-size 5m \
        --evaluation-frequency 1m \
        --action '$AG_ID' \
        > /tmp/az-app-alert.log 2>&1
    "
    ```
    Check: `cat /tmp/az-app-alert.log | tail -3`
  </instructions>
  <resume-signal>Type "alerts-wired" when Action Group and both alert rules are created.</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 3: Wire Grafana Cloud alert for Function App</name>
  <action>Connect Grafana Cloud to Azure and create alert pointing to /webhooks/grafana.</action>
  <instructions>
    **Grafana Cloud (free tier — https://grafana.com/auth/sign-up):**

    1. In Grafana Cloud → Connections → Azure Monitor:
       - Add datasource: type = Azure Monitor
       - Auth: Service Principal (use same pagemenot-azure SP credentials)
       - Subscription: your Azure subscription ID
       - Click Save & Test — should show green

    2. Create a Grafana alert rule:
       - Grafana → Alerting → Alert Rules → New alert rule
       - Name: `pagemenot-func-health`
       - Query: Azure Monitor → pagemenot-rg → Function App (pagemenot-test-func) → FunctionExecutionCount
       - Condition: `IS BELOW 1` for 5 minutes (or use "no data" as trigger)
       - Labels: add `alertname=Azure Function App Unhealthy`, `resource=pagemenot-test-func`
       - Contact point: (create new) → Webhook → URL: `https://34.123.60.64:8080/webhooks/grafana` → Send on: Alerting, OK
       - Save and enable

    3. Verify contact point works:
       - Grafana → Alerting → Contact Points → Test (pagemenot-grafana-webhook)
       - Check `docker logs pagemenot` on VM for the test POST arriving

    The Grafana payload will hit `/webhooks/grafana`. pagemenot's existing Grafana handler detects
    `cloud_provider=azure` from the alert name ("Azure Function App") and routes to the Azure runbook.
  </instructions>
  <resume-signal>Type "grafana-wired" when datasource connected and contact point test confirmed in VM logs.</resume-signal>
