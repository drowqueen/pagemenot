<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Provision Azure free-tier resources and service principal</name>
  <action>Azure account and service principal provisioning — must be done by user in Azure Portal / az CLI.</action>
  <instructions>
    This requires an Azure account (free tier: https://azure.microsoft.com/free/). Credit card required even on free tier.

    Once logged in to Azure CLI locally (`az login`), run these commands:

    **1. Create resource group:**
    ```bash
    az group create --name pagemenot-rg --location eastus
    ```

    **2. Create service principal with Contributor role:**
    ```bash
    az ad sp create-for-rbac \
      --name pagemenot-azure \
      --role Contributor \
      --scopes /subscriptions/$(az account show --query id -o tsv)/resourceGroups/pagemenot-rg \
      --output json
    ```
    Save the output: appId (→ AZURE_CLIENT_ID), password (→ AZURE_CLIENT_SECRET), tenant (→ AZURE_TENANT_ID).

    **3. Create B1s Linux VM:**
    ```bash
    az vm create \
      --resource-group pagemenot-rg \
      --name pagemenot-test-vm \
      --image Ubuntu2204 \
      --size Standard_B1s \
      --admin-username azureuser \
      --generate-ssh-keys \
      --public-ip-sku Standard
    ```

    **4. Create alert rule on VM heartbeat (Metric: Heartbeat, absent > 5min):**
    In Azure Portal → Monitor → Alerts → Create alert rule:
    - Scope: pagemenot-test-vm
    - Condition: Heartbeat count = 0 (absence) OR use "VM availability" metric < 1
    - Severity: Sev1
    - Action Group: (create new) → Action type: Webhook → URL: `https://34.123.60.64:8080/webhooks/azure`
    - Enable "Use common alert schema" (REQUIRED — toggle must be ON)

    **5. Update .env on the pagemenot VM:**
    ```bash
    gcloud compute ssh pagemenot --zone=us-central1-a --project=zipintel --command="
      echo 'AZURE_TENANT_ID=<tenant>' >> /home/leona/pagemenot/.env &&
      echo 'AZURE_CLIENT_ID=<appId>' >> /home/leona/pagemenot/.env &&
      echo 'AZURE_CLIENT_SECRET=<password>' >> /home/leona/pagemenot/.env &&
      echo 'AZURE_SUBSCRIPTION_ID=$(az account show --query id -o tsv)' >> /home/leona/pagemenot/.env
    "
    ```
    Then restart the container to pick up the new env vars:
    ```bash
    gcloud compute ssh pagemenot --zone=us-central1-a --project=zipintel --command="
      cd /home/leona/pagemenot && docker compose up -d
    "
    ```

    After restart, confirm az login succeeded:
    ```bash
    gcloud compute ssh pagemenot --zone=us-central1-a --project=zipintel --command="
      docker logs pagemenot 2>&1 | grep -i azure
    "
    ```
    Expected: `Azure CLI authenticated via service principal`
  </instructions>
  <resume-signal>Type "provisioned" when all five steps above are complete and az login confirmed in container logs.</resume-signal>
</task>
